## 🐛 BUGS A RESOLVER (PASO A PASO)

### 1. Pantalla Blanca en el Calendario (Crash de UI)
**Problema:** Al ingresar a la pestaña del calendario/turnos, la aplicación se rompe por completo (pantalla blanca).
**Solución Requerida:**
- Revisa `src/app/components/modules/AppointmentsModule.tsx` y los componentes hijos de calendario/turnos[cite: 3].
- Esto suele ser un error de renderizado en React (por ejemplo, intentar hacer `.map()` sobre un `undefined`, un error al parsear fechas de Firestore a JavaScript, o variables nulas). 
- Añade validaciones (ej. `if (!turnos) return <Loading />`) y asegúrate de que las fechas se manejen correctamente. Si hay un `ErrorBoundary`, verifica qué lo está disparando. Arregla también lo pendiente de la eliminación del campo "motivo de consulta".

### 2 y 3. Sincronización Global de Clientes y Mascotas en Comboboxes
**Problema:** Al registrar un nuevo Cliente, no aparece al querer registrar una Mascota. En la Historia Clínica, tampoco cargan los clientes ni las mascotas en los selectores.
**Solución Requerida:**
- Abre `src/app/components/modules/ClientsModule.tsx`, `PetsModule.tsx` (o su versión activa) y `MedicalHistoryModuleNew.tsx`[cite: 3].
- Asegúrate de que los `Combobox` / `Select` de Clientes y Mascotas estén consumiendo los datos directamente desde los hooks/servicios que leen Firestore (`clienteService.ts`, `mascotaService.ts`)[cite: 3].
- Usa `useEffect` o listeners en tiempo real (`onSnapshot`) para que, apenas se registre un cliente o mascota, los selectores de toda la app se actualicen con los datos más recientes.

### 4. Parámetros Opcionales y Registro de Profesionales
**Problema A:** En la sección de Parámetros, el sistema obliga a ingresar una descripción para cada cosa (ej. raza). 
**Solución A:** Revisa `src/app/components/modules/ParametrosModule.tsx` y el esquema de validación (Zod o similar)[cite: 3]. Haz que el campo `descripcion` sea **opcional** (`.optional()` o validación manual) tanto en la UI como en el tipado de TypeScript.
**Problema B:** No deja registrar nuevos profesionales.
**Solución B:** Revisa el formulario de creación de profesionales y `src/app/services/doctorService.ts`[cite: 3]. Asegúrate de que el payload enviado a Firebase coincida con las reglas de la base de datos y que los campos solicitados en el formulario sean estrictamente los necesarios.

### 5. Error Crítico de Login / Autenticación de Nuevos Usuarios
**Problema:** Se crea un usuario nuevo con email, se guarda con los roles correctos en la base de datos, pero el usuario no puede iniciar sesión.
**Solución Requerida:**
- Este es un error clásico de Firebase. Revisa `src/app/services/usuarioService.ts` y `AuthContext.tsx`[cite: 3].
- Cuando un administrador crea un usuario desde el panel usando `createUserWithEmailAndPassword`, Firebase automáticamente "desloguea" al admin e inicia sesión con el nuevo usuario, o falla si no se maneja bien.
- Verifica si la creación del usuario en Auth se está haciendo correctamente o si solo se está guardando el documento en Firestore sin crear la credencial real en Firebase Auth. Si el login falla, asegúrate de que el servicio de Login esté leyendo bien el estado de Auth y comparándolo con el documento de roles en Firestore.

### 6. Vacunas Inexistentes en Historia Clínica
**Problema:** Al agregar una vacuna en la Historia Clínica, dice que "no hay vacuna para esa mascota", a pesar de estar registrada.
**Solución Requerida:**
- Revisa la lógica en `MedicalHistoryModuleNew.tsx`[cite: 3] o donde se manejen las vacunas aplicadas.
- Verifica cómo se están filtrando las vacunas. Asegúrate de que la query que busca las vacunas de esa mascota esté comparando correctamente el `mascotaId` con los registros existentes en la base de datos.
- Corrige el filtro para que el Combobox de vacunas muestre el historial correcto asociado a la mascota seleccionada.
